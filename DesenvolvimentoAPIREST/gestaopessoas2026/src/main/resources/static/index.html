Gestão Pessoas 2026

API REST desenvolvida com Spring Boot

Sobre o projeto

Esta aplicação simula o cadastro, consulta, atualização e remoção de pessoas, utilizando uma arquitetura em camadas com Controller, Service, Model e tratamento centralizado de exceções.

Endpoints principais

GET /pessoas

Retorna a lista de pessoas cadastradas.

GET /pessoas/1

Busca uma pessoa pelo ID informado.

POST /pessoas

Cria uma nova pessoa a partir de um JSON enviado no corpo da requisição.

PUT /pessoas

Atualiza os dados de uma pessoa existente usando JSON no corpo da requisição.

DELETE /pessoas/{id}

Remove uma pessoa pelo ID informado.

Testes de exceções

GET /teste/int/0

Simula uma exceção genérica, retornando erro interno do servidor.

GET /teste/bad/-1

Simula uma requisição inválida, retornando Bad Request.

GET /teste/nao/401

Simula acesso não autorizado, retornando Unauthorized.

Exemplo de JSON para POST

{
  "nome": "Guilherme",
  "sobrenome": "Carrijo",
  "endereco": "Morrinhos - GO",
  "genero": "Masculino"
}

  "id": 1,
